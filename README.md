## HTTPS for mobile development

Modern Android browsers require a secure context (HTTPS) to access device cameras via `navigator.mediaDevices.getUserMedia()`.

For local development, HTTPS can be enabled using a locally trusted CA generated by `mkcert`.

### Install mkcert

Ubuntu:

```bash
sudo apt install libnss3-tools
```

Install mkcert according to its documentation.

Initialize local CA:

```bash
mkcert -install
```

Verify CA location:

```bash
mkcert -CAROOT
```

Expected files:

```text
rootCA.pem
rootCA-key.pem
```

### Generate certificate for local server

Example:

```bash
mkcert 192.168.0.136 localhost
```

Generated files:

```text
192.168.0.136+1.pem
192.168.0.136+1-key.pem
```

### Configure React development server

Create `.env` file inside the web client project:

```text
HTTPS=true
SSL_CRT_FILE=./certs/192.168.0.136+1.pem
SSL_KEY_FILE=./certs/192.168.0.136+1-key.pem
```

Or start manually:

```bash
HTTPS=true \
SSL_CRT_FILE=./certs/192.168.0.136+1.pem \
SSL_KEY_FILE=./certs/192.168.0.136+1-key.pem \
npm start
```

### Install local CA on Android

Copy:

```text
rootCA.pem
```

to the Android device.

Rename if necessary:

```text
rootCA.crt
```

Install:

```text
Settings
→ Security
→ Install certificate
→ CA certificate
```

Verify that the certificate appears under:

```text
Settings
→ Security
→ Trusted credentials
→ User credentials
```

Expected certificate name:

```text
mkcert development CA
```

### Access application

Example:

```text
https://192.168.0.136:3000
```

### Notes

* HTTPS is required for camera access.
* Chrome on Android is the reference browser for testing.
* Some browsers (for example DuckDuckGo Browser) may reject locally trusted certificates even when Android trusts them.
* This setup is intended only for development.
* Production deployment should use a publicly trusted CA (e.g. Let's Encrypt).

## QR / Barcode Scanning

The system supports QR code and barcode scanning from mobile devices.

### Supported platforms

Tested and supported:

* Android 13+
* Google Chrome
* Chromium

Not tested or unsupported:

* DuckDuckGo Browser
* Opera Mini
* Browsers with custom certificate validation logic

### HTTPS requirement

Camera access requires HTTPS.

For local development, HTTPS is provided using a local CA generated with `mkcert`.

See the **HTTPS for mobile development** section for details.

### Scanning workflow

1. User opens Dashboard.
2. User presses Scan button.
3. Camera view opens.
4. QR code or barcode is scanned.
5. Frontend calls:

```http
GET /api/v1/machine/barcode/{barcode}
```

6. If a machine is found:

```text
/machines/{id}
```

page is opened automatically.

7. If no machine is found:

* show notification;
* remain on Dashboard.

### Development notes

* The scanner attempts to use the device rear camera.
* Rear camera selection depends on browser and device capabilities.
* If rear camera cannot be identified, the first available camera is used.
* Camera access is only guaranteed on supported browsers.

### Known limitations

* Browser camera labels are vendor-specific.
* Some Android devices expose multiple rear cameras.
* Camera ordering is not standardized between devices.


## Known issues

* Under VPN, Docker dependencies and npm packages may work unstably.
* Mobile camera access requires HTTPS.
* Some Android browsers may not trust locally generated certificates.
